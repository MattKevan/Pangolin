# Plan: Change Startup Flow to Load App Shell First

## Current Startup Flow

**Entry point**: `PangolinApp.swift` (line 8)

The app uses a `ZStack` in the `WindowGroup`:
1. `MainView` (the app shell) is always rendered underneath
2. `StartupOverlayView` covers it when `libraryManager.isLibraryOpen == false`
3. On `.onAppear`, `startLibraryStartup()` calls `libraryManager.smartStartup()` which creates/opens the library
4. `MainView` has `.allowsHitTesting(libraryManager.isLibraryOpen)` which disables all interaction while loading

**StartupOverlayView** (line 187): Full-screen overlay with dark blur background and a card showing either:
- Loading: ProgressView + "Opening Library..." + progress bar
- Error: "Couldn't Open Library" + retry/reset buttons

**Problem**: The user sees a loading screen instead of the app shell. The shell is rendered but hidden behind the overlay and non-interactive.

---

## Proposed Change

### 1. Remove the StartupOverlayView overlay (`PangolinApp.swift`)

- Delete the `StartupOverlayView` from the `ZStack` (lines 23-31 for macOS, lines 99-107 for iOS)
- Remove `.allowsHitTesting(libraryManager.isLibraryOpen)` from `MainView` (lines 21, 97)
- Keep the `.onAppear` startup logic and the animation

### 2. Add inline loading state to MainView

- Add a banner or indicator at the top of `MainView` when `!libraryManager.isLibraryOpen`
- Show a small ProgressView + "Opening Library..." text in the toolbar or as a top banner
- The sidebar's smart collections section renders immediately (doesn't depend on library)
- The "Library" section in sidebar shows empty/loading while library loads

### 3. Show loading state in detail area

- When `libraryManager.isLibraryOpen == false` and no video is selected, show a loading state in the detail column instead of "No video selected"
- Could be a centered ProgressView with "Opening Library..." text

### 4. Handle errors inline

- When `libraryManager.error` is set, show an error state in the detail area or as an alert
- Keep the retry/reset functionality accessible from the menu bar (already exists: Cmd+Shift+R for Reload Library)

### LibraryLoadingBanner (new component)

A small, non-intrusive banner at the top of the window:

```swift
struct LibraryLoadingBanner: View {
    let progress: Double
    
    var body: some View {
        HStack {
            ProgressView()
                .controlSize(.small)
            Text("Opening Library...")
            if progress > 0 {
                ProgressView(value: progress)
                    .progressViewStyle(.linear)
                    .frame(width: 120)
            }
        }
        .padding(.horizontal, 16)
        .padding(.vertical, 8)
        .background(.ultraThinMaterial)
    }
}
```
